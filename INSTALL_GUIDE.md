# 설치 가이드 (완전 초보자용)

컴퓨터 프로그램에 익숙하지 않은 분들도 따라할 수 있도록 만든 안내서입니다.
각 단계의 스크린을 보면서 **그대로 따라하시면** 됩니다.

---

## 이 프로그램은 무엇인가요?

AI(Claude, Cursor 등)에게 **"민법 검색해줘"**, **"부당해고 판례 찾아줘"** 라고 말하면
법제처 국가법령정보센터에서 법령과 판례를 자동으로 찾아주는 프로그램입니다.

이 프로그램을 사용하려면 아래 4가지를 준비해야 합니다:

| 순서 | 할 일 | 소요 시간 |
|------|--------|-----------|
| 1 | Git 설치 | 5분 |
| 2 | Node.js 설치 | 5분 |
| 3 | 프로젝트 다운로드 및 빌드 | 5분 |
| 4 | 법제처 API 키 발급 | 10~30분 |
| 5 | AI 앱에 연결 설정 | 5분 |

**총 약 30분~1시간** 이면 모든 설정이 완료됩니다.

---

## 먼저 알아야 할 것: 터미널(명령 프롬프트)

설치 과정에서 **터미널**이라는 프로그램을 사용합니다.
터미널은 컴퓨터에게 글자로 명령을 내리는 프로그램입니다.

### 터미널 여는 방법

**Windows:**

1. 키보드에서 `Windows 키`(깃발 모양)와 `R` 키를 **동시에** 누릅니다.
2. 작은 창이 뜨면 `cmd` 라고 입력합니다.
3. `Enter` 키를 누릅니다.
4. 검은색(또는 흰색) 글자 입력 화면이 나타나면 성공입니다.

**macOS:**

1. 키보드에서 `Command(⌘)` 키와 `Space(스페이스바)` 키를 **동시에** 누릅니다.
2. 검색창이 뜨면 `터미널` 이라고 입력합니다.
3. `터미널.app`이 검색되면 `Enter` 키를 누릅니다.
4. 흰색(또는 검은색) 글자 입력 화면이 나타나면 성공입니다.

### 터미널 사용법 기초

```
이렇게 생긴 회색 박스 안의 글자가 "명령어"입니다.
이 명령어를 터미널에 그대로 입력하고 Enter를 누르면 됩니다.
```

- 명령어는 **한 줄씩** 입력하고 **Enter**를 눌러 실행합니다.
- `#`으로 시작하는 줄은 설명이므로 입력하지 않아도 됩니다.
- 명령어 입력 후 아무 반응 없이 다음 줄이 나타나면 **정상적으로 실행된 것**입니다.
- 빨간색 글자로 `error`가 나타나면 문제가 생긴 것입니다. [FAQ](#자주-묻는-질문-faq) 섹션을 참고하세요.

## 1단계: Git 설치

### Git이란?

인터넷에 올라가 있는 프로그램 소스코드를 내 컴퓨터로 다운로드하는 도구입니다.
일반적인 파일 다운로드와 비슷하지만, 프로그래머들이 사용하는 전문 도구입니다.

### Windows에서 Git 설치하기

**1.** 인터넷 브라우저(Chrome, Edge 등)에서 아래 주소로 접속합니다:

```
https://git-scm.com/downloads/win
```

**2.** 페이지에서 **"Click here to download"** 라는 링크를 클릭합니다.

**3.** 파일이 다운로드되면, 다운로드된 파일을 **더블클릭**하여 실행합니다.
  - 보통 `Git-2.xx.x-64-bit.exe` 같은 이름입니다.
  - "이 앱이 변경하도록 허용하시겠습니까?" 라고 물으면 **"예"** 를 누릅니다.

**4.** 설치 화면이 나타나면:
  - **아무것도 변경하지 말고** 계속 **"Next"** 버튼만 누릅니다.
  - 마지막에 **"Install"** 버튼을 누릅니다.
  - 설치가 끝나면 **"Finish"** 버튼을 누릅니다.

**5.** 설치가 잘 되었는지 확인합니다:
  - 터미널을 **새로** 엽니다. (이미 열려있던 터미널은 닫고 다시 열어야 합니다)
  - 아래 명령어를 입력하고 Enter를 누릅니다:

```bash
git --version
```

**6.** 아래와 비슷한 결과가 나오면 **설치 성공**입니다:

```
git version 2.47.1
```

> 숫자는 다를 수 있습니다. `git version` 으로 시작하는 글자가 나오면 OK입니다.

### macOS에서 Git 설치하기

macOS에는 Git이 이미 포함되어 있는 경우가 많습니다.

**1.** 터미널을 엽니다.

**2.** 아래 명령어를 입력하고 Enter를 누릅니다:

```bash
git --version
```

**3.** 두 가지 경우 중 하나가 됩니다:

- **버전이 표시되는 경우** (예: `git version 2.39.0`) → 이미 설치되어 있습니다. 다음 단계로 넘어가세요.
- **설치 팝업이 뜨는 경우** → **"설치"** 버튼을 누르고 완료될 때까지 기다립니다. (약 5분)

## 2단계: Node.js 설치

### Node.js란?

이 프로그램을 실행하기 위해 필요한 도구입니다.
Node.js가 없으면 이 프로그램이 동작하지 않습니다.

### Windows / macOS 공통

**1.** 인터넷 브라우저에서 아래 주소로 접속합니다:

```
https://nodejs.org
```

**2.** 화면에 큰 초록색 버튼이 두 개 보입니다.
  - **왼쪽 버튼 "LTS"** 를 클릭합니다. (LTS = 안정적인 버전이라는 뜻)
  - 오른쪽 "Current" 버튼은 누르지 마세요.

**3.** 파일이 다운로드되면, 다운로드된 파일을 **더블클릭**하여 실행합니다.

**4.** 설치 화면이 나타나면:
  - **아무것도 변경하지 말고** 계속 **"Next"** (또는 **"계속"**) 버튼만 누릅니다.
  - 약관 동의에 체크하고 진행합니다.
  - 마지막에 **"Install"** (또는 **"설치"**) 버튼을 누릅니다.
  - Windows에서 "이 앱이 변경하도록 허용하시겠습니까?" 라고 물으면 **"예"** 를 누릅니다.
  - 설치가 끝나면 **"Finish"** (또는 **"닫기"**) 버튼을 누릅니다.

**5.** 설치가 잘 되었는지 확인합니다:
  - 터미널을 **새로** 엽니다. (이미 열려있던 터미널은 닫고 다시 열어야 합니다)
  - 아래 명령어를 **한 줄씩** 입력하고 각각 Enter를 누릅니다:

```bash
node --version
```

```bash
npm --version
```

**6.** 각각 버전 번호가 나오면 **설치 성공**입니다:

```
v22.14.0
10.9.2
```

> 숫자는 다를 수 있습니다. 숫자가 나오면 OK입니다.
> `command not found` 또는 `'node'은(는) 내부 또는 외부 명령...이 아닙니다` 라고 나오면 설치가 안 된 것입니다. 위의 과정을 다시 시도해 주세요.

## 3단계: 프로젝트 다운로드 및 빌드

### 3-1. 다운로드할 폴더 정하기

프로젝트를 저장할 위치를 정합니다. 여기서는 **문서(Documents)** 폴더에 저장하겠습니다.

터미널에 아래 명령어를 입력합니다:

**Windows:**
```bash
cd %USERPROFILE%\Documents
```

**macOS:**
```bash
cd ~/Documents
```

> **`cd`란?** "change directory"의 줄임말로, 폴더를 이동하는 명령어입니다.
> 다른 폴더에 저장하고 싶다면 해당 경로로 변경하면 됩니다.

### 3-2. 프로젝트 다운로드하기 (git clone)

아래 명령어를 입력하고 Enter를 누릅니다:

```bash
git clone https://github.com/hjsh200219/public-data-mcp.git
```

> **이 명령어가 하는 일:**
> GitHub(프로그래머들의 코드 저장소)에 올라가 있는 이 프로젝트를
> 내 컴퓨터의 Documents 폴더 안에 `public-data-mcp`라는 폴더로 복사합니다.

아래와 비슷한 메시지가 나오면 성공입니다:

```
Cloning into 'public-data-mcp'...
remote: Enumerating objects: 30, done.
...
Resolving deltas: 100% (10/10), done.
```

### 3-3. 프로젝트 폴더로 이동

```bash
cd public-data-mcp
```

### 3-4. 필요한 패키지 설치 (npm install)

```bash
npm install
```

> **이 명령어가 하는 일:**
> 프로그램 실행에 필요한 부품(라이브러리)들을 인터넷에서 자동으로 다운로드합니다.
> 1~2분 정도 걸리며, 여러 줄의 텍스트가 빠르게 지나가는 것은 정상입니다.

완료되면 아래와 비슷한 메시지가 나옵니다:

```
added 50 packages in 10s
```

### 3-5. 프로젝트 빌드하기 (npm run build)

```bash
npm run build
```

> **이 명령어가 하는 일:**
> 소스코드를 컴퓨터가 실행할 수 있는 형태로 변환합니다.
> 몇 초 안에 완료되며, 오류 없이 끝나면 성공입니다.

**성공 확인 방법:**
- 빨간색 에러 메시지 없이 명령어가 끝나면 성공입니다.
- 프로젝트 폴더 안에 `dist` 라는 새 폴더가 생긴 것을 확인할 수 있습니다.

### 3-6. 프로젝트 경로 확인 (나중에 필요합니다!)

이 프로젝트가 어디에 저장되었는지 **경로**를 확인해 둡니다.
이 경로는 5단계에서 사용합니다.

**macOS:**
```bash
pwd
```

**Windows:**
```bash
cd
```

아래와 같은 결과가 나옵니다. 이것을 **메모장에 복사해 두세요**:

```
macOS 예시:   /Users/내이름/Documents/public-data-mcp
Windows 예시: C:\Users\내이름\Documents\public-data-mcp
```

## 4단계: 법제처 API 키 발급

### API 키란?

법제처 서버에 "저는 허가받은 사용자입니다"라고 증명하는 **인증 코드**입니다.
아파트 출입카드처럼, 이 코드가 있어야 법령 데이터를 가져올 수 있습니다.
**무료**로 발급받을 수 있습니다.

### 4-1. 법제처 오픈API 사이트 접속

인터넷 브라우저에서 아래 주소로 접속합니다:

```
https://open.law.go.kr/LSO/openApi/guideList.do
```

### 4-2. 회원가입

1. 화면 우측 상단의 **"회원가입"** 버튼을 클릭합니다.
2. 가입 유형을 선택합니다. (개인 사용자로 가입하면 됩니다)
3. 이름, 이메일, 비밀번호 등 필요한 정보를 입력합니다.
4. 약관에 동의하고 **가입 완료** 버튼을 누릅니다.

### 4-3. 로그인

1. 우측 상단의 **"로그인"** 버튼을 클릭합니다.
2. 가입할 때 입력한 아이디와 비밀번호로 로그인합니다.

### 4-4. API 인증코드 신청

1. 로그인 후 **"오픈API 이용안내"** 메뉴로 이동합니다.
2. **"오픈API 신청"** 버튼을 클릭합니다.
3. 신청 양식을 작성합니다:
   - **활용 목적**: "법령 정보 검색 및 학습용" 등으로 자유롭게 작성합니다.
   - 기타 항목도 적절히 채워 넣습니다.
4. **"신청"** 버튼을 누릅니다.

### 4-5. 인증코드 확인

- 신청 후 승인까지 **즉시 ~ 수 시간** 정도 걸릴 수 있습니다.
- 승인되면 마이페이지에서 **인증코드(OC)** 를 확인할 수 있습니다.
- 인증코드는 영문과 숫자로 이루어진 문자열입니다.

```
인증코드 예시: a1b2c3d4e5f6g7h8
```

> 이 인증코드를 **메모장에 복사해 두세요**. 다음 단계에서 사용합니다.

## 5단계: AI 앱에 연결 설정 (MCP 설정)

### MCP란?

**MCP(Model Context Protocol)** 는 AI 앱(Claude, Cursor 등)에
외부 도구를 연결하는 규격입니다.
쉽게 말해, AI에게 "법령 검색 기능"을 추가하는 **플러그인**이라고 생각하시면 됩니다.

이 단계에서는 AI 앱의 설정 파일에 이 프로그램의 정보를 적어줍니다.
사용하는 앱에 따라 **하나만** 따라하시면 됩니다.

---

### 방법 A: Claude Desktop을 사용하는 경우

#### A-1. 설정 파일 위치 찾기

**macOS:**

1. Finder(파일 탐색기)를 엽니다.
2. 상단 메뉴에서 **"이동" → "폴더로 이동"** 을 클릭합니다 (또는 `Cmd + Shift + G`).
3. 아래 경로를 붙여넣고 Enter를 누릅니다:

```
~/Library/Application Support/Claude
```

4. `Claude` 폴더가 열립니다.
5. 이 폴더 안에 `claude_desktop_config.json` 파일이 있는지 확인합니다.
   - **있으면**: 이 파일을 텍스트 편집기(메모장 등)로 엽니다.
   - **없으면**: 새 텍스트 파일을 만들어서 이름을 `claude_desktop_config.json`으로 저장합니다.

**Windows:**

1. 파일 탐색기를 엽니다 (`Windows 키 + E`).
2. 주소 표시줄에 아래 경로를 붙여넣고 Enter를 누릅니다:

```
%APPDATA%\Claude
```

3. `Claude` 폴더가 열립니다.
4. 이 폴더 안에 `claude_desktop_config.json` 파일이 있는지 확인합니다.
   - **있으면**: 이 파일을 마우스 오른쪽 클릭 → **"연결 프로그램" → "메모장"** 으로 엽니다.
   - **없으면**: 폴더 안에서 마우스 오른쪽 클릭 → **"새로 만들기" → "텍스트 문서"** → 이름을 `claude_desktop_config.json`으로 변경합니다.

> **주의:** Windows에서 파일 확장자가 보이지 않으면 `.json.txt` 같은 잘못된 이름이 될 수 있습니다.
> 파일 탐색기 상단의 **"보기" → "파일 확장명"** 에 체크하여 확장자가 보이게 하세요.

#### A-2. 설정 내용 입력

파일을 열고, 아래 내용을 **통째로 복사해서 붙여넣기** 합니다.

> **처음 만든 빈 파일인 경우** — 아래 내용을 그대로 붙여넣습니다:

```json
{
  "mcpServers": {
    "public-data": {
      "command": "node",
      "args": ["여기에_프로젝트_경로_입력/dist/index.js"],
      "env": {
        "LAW_API_OC": "여기에_인증코드_입력"
      }
    }
  }
}
```

> **이미 다른 MCP 설정이 있는 경우** — `"mcpServers": {` 중괄호 안에 아래 부분만 추가합니다.
> 기존 설정 마지막 `}` 뒤에 쉼표(`,`)를 꼭 넣어주세요:

```json
    "public-data": {
      "command": "node",
      "args": ["여기에_프로젝트_경로_입력/dist/index.js"],
      "env": {
        "LAW_API_OC": "여기에_인증코드_입력"
      }
    }
```

#### A-3. 경로와 인증코드 수정하기

**두 곳**을 수정해야 합니다:

**1) 프로젝트 경로 수정**

`"여기에_프로젝트_경로_입력/dist/index.js"` 부분을 3단계에서 메모해 둔 경로로 바꿉니다.

macOS 예시:
```json
"args": ["/Users/내이름/Documents/public-data-mcp/dist/index.js"],
```

Windows 예시 (역슬래시를 두 개씩 써야 합니다):
```json
"args": ["C:\\Users\\내이름\\Documents\\public-data-mcp\\dist\\index.js"],
```

**2) 인증코드 수정**

`"여기에_인증코드_입력"` 부분을 4단계에서 메모해 둔 인증코드로 바꿉니다.

```json
"LAW_API_OC": "a1b2c3d4e5f6g7h8"
```

#### A-4. 완성된 설정 파일 예시

**macOS 사용자 완성 예시:**
```json
{
  "mcpServers": {
    "public-data": {
      "command": "node",
      "args": ["/Users/홍길동/Documents/public-data-mcp/dist/index.js"],
      "env": {
        "LAW_API_OC": "a1b2c3d4e5f6g7h8"
      }
    }
  }
}
```

**Windows 사용자 완성 예시:**
```json
{
  "mcpServers": {
    "public-data": {
      "command": "node",
      "args": ["C:\\Users\\홍길동\\Documents\\public-data-mcp\\dist\\index.js"],
      "env": {
        "LAW_API_OC": "a1b2c3d4e5f6g7h8"
      }
    }
  }
}
```

#### A-5. 저장 및 재시작

1. 파일을 **저장**합니다 (`Ctrl + S` 또는 `Cmd + S`).
2. Claude Desktop 앱을 **완전히 종료**합니다.
   - macOS: Dock에서 오른쪽 클릭 → "종료"
   - Windows: 시스템 트레이(화면 우측 하단)에서 Claude 아이콘 오른쪽 클릭 → "종료"
3. Claude Desktop을 **다시 실행**합니다.

## 자주 묻는 질문 (FAQ)

### 터미널 관련

**Q. 터미널에 명령어를 입력했는데 아무 일도 안 일어납니다.**

정상입니다! 대부분의 명령어는 성공하면 아무 메시지도 출력하지 않습니다.
다음 줄에 입력 커서가 나타나면 정상적으로 완료된 것입니다.

**Q. `'xxx'은(는) 내부 또는 외부 명령, 실행할 수 있는 프로그램, 또는 배치 파일이 아닙니다.` 라고 나옵니다.**

해당 프로그램이 설치되지 않았거나, 터미널을 새로 열지 않아서 발생합니다.
- 프로그램을 설치한 뒤에는 반드시 터미널을 **닫고 새로 열어야** 합니다.
- 프로그램 설치가 제대로 되었는지 다시 확인하세요.

**Q. macOS에서 `command not found` 라고 나옵니다.**

위와 같은 원인입니다. 터미널을 닫고 새로 열어 보세요.

### Git 관련

**Q. `git clone`이 안 됩니다.**

- 인터넷에 연결되어 있는지 확인하세요.
- Git이 설치되었는지 `git --version`으로 확인하세요.
- 회사/학교 네트워크에서는 방화벽 때문에 차단될 수 있습니다. 개인 네트워크(집 Wi-Fi, 모바일 핫스팟)로 시도해 보세요.

### Node.js / npm 관련

**Q. `npm install`에서 오류가 납니다.**

- Node.js가 설치되었는지 `node --version`으로 확인하세요.
- Node.js 18 이상이 필요합니다. 버전이 낮으면 https://nodejs.org 에서 LTS를 다시 설치하세요.
- 인터넷 연결을 확인하세요.

**Q. `npm run build`에서 오류가 납니다.**

- `npm install`을 먼저 실행했는지 확인하세요.
- 프로젝트 폴더(`public-data-mcp`) 안에서 실행하고 있는지 확인하세요.
  터미널에 `pwd` (macOS) 또는 `cd` (Windows)를 입력하면 현재 위치를 알 수 있습니다.

### API 키 관련

**Q. API 키를 어디서 확인하나요?**

법제처 오픈API 사이트(https://open.law.go.kr)에 로그인 후
마이페이지에서 발급받은 인증코드(OC)를 확인할 수 있습니다.

**Q. API 키를 넣었는데 검색이 안 됩니다.**

- 인증코드 앞뒤에 **공백(빈칸)이 없는지** 확인하세요.
- 인증코드를 따옴표(`"`) 안에 넣었는지 확인하세요.
- 법제처 API 신청 후 **승인까지 시간이 걸릴 수 있습니다**. 잠시 기다려 보세요.

### MCP 설정 관련

**Q. Claude Desktop / Cursor에서 MCP 도구가 보이지 않습니다.**

- 설정 파일(JSON)의 **경로가 정확한지** 다시 확인하세요. 오타가 가장 흔한 원인입니다.
- 앱을 **완전히 종료 후 재시작** 했는지 확인하세요.
- JSON 파일에 **문법 오류**가 없는지 확인하세요. 흔한 실수:
  - 쉼표(`,`)를 빠뜨리거나 잘못 넣은 경우
  - 중괄호(`{`, `}`)나 대괄호(`[`, `]`)의 짝이 안 맞는 경우
  - 따옴표(`"`)를 빠뜨린 경우

**Q. JSON 문법이 맞는지 확인하는 방법이 있나요?**

아래 무료 사이트에 설정 파일 내용을 붙여넣으면 문법 오류를 자동으로 찾아줍니다:

```
https://jsonlint.com
```

## 용어 정리

| 용어 | 설명 |
|------|------|
| **터미널** | 컴퓨터에게 글자로 명령을 내리는 프로그램. Windows에서는 "명령 프롬프트"라고도 합니다. |
| **Git** | 프로그래머들이 코드를 공유하고 관리하는 도구. 여기서는 코드를 다운로드하는 용도로 사용합니다. |
| **git clone** | Git을 이용해 인터넷에 있는 프로젝트를 내 컴퓨터에 복사(다운로드)하는 명령어. |
| **GitHub** | 프로그래머들이 코드를 저장하고 공유하는 웹사이트. 이 프로젝트가 올라가 있는 곳입니다. |
| **Node.js** | JavaScript라는 프로그래밍 언어를 실행해주는 도구. 이 프로그램을 실행하는 데 필요합니다. |
| **npm** | Node.js의 패키지(부품) 관리자. `npm install`로 필요한 부품을 설치합니다. |
| **npm install** | 프로젝트에 필요한 라이브러리(부품)를 인터넷에서 자동으로 다운로드하는 명령어. |
| **npm run build** | 소스코드를 실행 가능한 형태로 변환(빌드)하는 명령어. |
| **API** | Application Programming Interface. 프로그램끼리 정보를 주고받는 방법. |
| **API 키** | API를 사용하기 위한 인증 코드. 비밀번호와 비슷한 역할을 합니다. |
| **MCP** | Model Context Protocol. AI 앱에 외부 도구를 연결하는 규격(플러그인 시스템). |
| **JSON** | 설정 정보를 저장하는 텍스트 형식. 중괄호 `{}`와 따옴표 `""`를 사용합니다. |
| **경로(Path)** | 컴퓨터에서 파일이나 폴더의 위치를 나타내는 주소. 예: `/Users/이름/Documents/파일.txt` |
| **LTS** | Long Term Support. 오랫동안 안정적으로 지원되는 버전이라는 뜻. |
